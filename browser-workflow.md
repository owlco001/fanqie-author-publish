# 番茄浏览器工作流

## 目标

这个文档定义了 `fanqie-author-publish` 技能在浏览器侧的执行清单。

它用于指导 OpenClaw 的浏览器自动化流程，确保步骤清晰、可验证、可中断。

## 前置条件

开始前必须满足：
- 用户已经登录番茄作者后台
- 目标章节内容已经从 JSON 文件或粘贴内容中准备好
- 当前浏览器会话使用的是正确的作者账号
- 当前没有验证码或风控提示

任何一个前置条件不满足，都要停止并报告原因。

## 必需输入

- `book_name`
- `chapter_title`
- `chapter_body`
- `action`

可选输入：
- `publish_time`
- `notes`
- `source_file`

## 工作流状态

- `pending`
- `backend_opened`
- `book_selected`
- `editor_opened`
- `draft_loaded`
- `draft_saved`
- `awaiting_confirmation`
- `published_verified`
- `failed_needs_review`

## 阶段 1：打开后台

1. 打开番茄作者后台地址。
2. 等待页面完全加载。
3. 确认当前页面不是登录页。
4. 如果页面可见账号信息，确认当前账号与预期作者账号一致。

成功条件：
- 已看到作者后台首页或工作台

失败条件：
- 出现登录页
- 出现验证码
- 后台页面加载失败

## 阶段 2：定位目标作品

1. 找到作品列表、书籍列表或创作管理入口。
2. 进入作品列表。
3. 搜索或人工可见定位 `book_name`。
4. 打开匹配作品的管理页。

成功条件：
- 当前作品页明确匹配 `book_name`

失败条件：
- 没有找到匹配作品
- 存在多个相似作品且无法明确区分

## 阶段 3：打开章节编辑器

1. 找到章节管理入口。
2. 打开章节列表或章节编辑页。
3. 根据 `action` 点击新建章节或编辑现有草稿。
4. 等待编辑器完全加载。

成功条件：
- 可以看到标题输入框和章节编辑区域

失败条件：
- 编辑器没有加载完成
- 编辑器处于只读状态
- 页面发生意外跳转

## 阶段 4：填写章节内容

1. 如有必要，先清空章节标题输入框。
2. 输入 `chapter_title`。
3. 聚焦章节正文编辑器。
4. 填入 `chapter_body`。
5. 短暂等待后确认正文确实显示在编辑器中。

重点检查：
- 标题输入框中是否为预期标题
- 编辑器中是否存在非空正文
- 正文是否出现截断或缺失

成功条件：
- 标题和正文都正确显示

失败条件：
- 标题没有成功写入
- 正文没有出现在编辑器中
- 正文内容部分缺失

## 阶段 5：保存草稿

这个阶段用于 `prepare_draft`，也用于任何最终发布前的预保存动作。

1. 定位“保存草稿”按钮。
2. 点击一次保存草稿。
3. 等待成功提示、状态标签、跳转结果或列表更新。
4. 截图保存。

成功条件：
- 出现草稿保存成功提示，或在章节列表中能看到草稿状态

失败条件：
- 保存按钮不可点击
- 出现错误提示
- 等待后没有任何状态变化

## 阶段 6：检查发布条件

用于 `review`，也用于 `publish` 之前的最终检查。

检查：
- `book_name` 正确
- `chapter_title` 正确
- 正文存在
- 如果平台显示草稿状态，草稿已存在
- 如果平台显示发布按钮，发布按钮状态正常

返回以下之一：
- `ready to publish`
- `not ready`

如果不能发布，必须明确指出阻塞项。

## 阶段 7：发布

只有满足以下条件时才进入此阶段：
- `action` 为 `publish`
- 用户在当前会话中已明确确认最终发布

步骤：
1. 再次确认 `book_name` 和 `chapter_title`。
2. 找到最终发布按钮。
3. 只点击一次发布。
4. 如果出现确认弹窗，核对文案后只确认一次。
5. 等待发布成功结果。
6. 截图保存。
7. 如果可以，返回章节列表并确认章节状态为已发布。

成功条件：
- 出现成功提示，或章节状态清楚显示为已发布

失败条件：
- 找不到发布按钮
- 确认弹窗内容异常
- 等待超时且没有成功状态

## 阶段 8：核对最近章节

当 `action` 为 `reconcile` 时，执行这个阶段。

1. 打开 `book_name` 对应的章节列表。
2. 读取最近章节标题和状态。
3. 与提供的本地章节列表进行比对。
4. 输出不一致项。

不一致类型包括：
- 平台缺少章节
- 标题重复
- 仍然是草稿
- 出现意外的额外章节

## 停止条件

出现以下任一情况时，必须立即停止：
- 登录页
- 验证码或人工校验
- 风控警告
- 账号不匹配
- 页面文案或结构与预期流程明显不一致

## 截图节点

在这些节点建议截图：
- 后台首页加载完成
- 目标作品页面打开
- 编辑器填写完成
- 草稿保存完成
- 发布成功或失败

## 最小输出约定

每次运行都应输出：
- `state`
- `book_name`
- `chapter_title`
- `action`
- `result`
- `warning`
- 截图状态

示例输出：

```text
state: draft_saved
book_name: 作品名
chapter_title: 第12章 标题示例
action: prepare_draft
result: 已成功保存草稿。
warning: 无。
```
