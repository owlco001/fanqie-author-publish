# 页面定位与选择策略

这个文件定义 `fanqie-author-publish` 在浏览器自动化中的页面定位策略。

目标不是写死某个 CSS 选择器，而是优先使用更稳的定位方法：
- 可见文字
- 页面区域语义
- 关键按钮名称
- 状态提示
- 后备定位策略

这样即使页面局部结构发生变化，仍然有机会保持可用。

## 总体原则

优先级从高到低：
1. 明确可见的按钮或标题文字
2. 明确的页面区域名称
3. 表单标签和输入框邻近关系
4. 状态提示文案
5. 结构性后备方法

不要优先依赖：
- 深层 CSS 路径
- 易变化的 class 名
- 第几个按钮这类位置假设
- 没有文案上下文的纯图标按钮

## 页面 1：作者后台首页

### 目标

确认已经进入番茄作者后台，而不是普通站点首页或登录页。

### 优先定位信号

- 页面上出现“作者后台”“创作平台”“工作台”“作品管理”这类后台入口文字
- 页面存在作品列表、创作管理、章节管理等后台功能区
- 页面右上或页面头部存在当前作者账号信息

### 成功判断

满足以下任意一项即可：
- 能看到“作品管理”或同类入口
- 能看到当前作者已登录后的工作台区域
- 能进入作品列表页

### 失败判断

- 出现“登录”“手机号登录”“验证码登录”等登录页文字
- 出现身份验证、验证码、风险提示

### 后备策略

- 从侧边栏查找与“创作”“作品”“章节”相关入口
- 从页面主区域查找卡片式的作品管理入口

## 页面 2：作品列表页

### 目标

找到目标作品 `book_name`。

### 优先定位信号

- 可见作品卡片
- 可见作品标题文字
- 页面存在“作品管理”“我的作品”“作品列表”等文案

### 定位策略

1. 优先在搜索框输入 `book_name`，如果页面有搜索能力
2. 如果没有搜索框，遍历可见作品卡片文字
3. 定位到与 `book_name` 完全一致的作品
4. 打开该作品的管理入口

### 成功判断

- 当前打开页面的作品标题与 `book_name` 一致

### 失败判断

- 没有找到目标作品
- 找到多个近似作品但无法唯一确认

### 后备策略

- 允许使用部分标题匹配，但必须在点击前再次确认完整作品名
- 如果存在“进入管理”“管理作品”“查看详情”等多个按钮，优先选与目标作品同一卡片内的按钮

## 页面 3：作品管理页

### 目标

从作品管理页进入章节相关功能。

### 优先定位信号

- 页面标题中出现作品名
- 页面存在“章节管理”“作品设置”“数据中心”等管理模块

### 定位策略

1. 在页面导航、侧边栏或页签中寻找“章节管理”
2. 如果没有完全一致文案，寻找“章节”“内容管理”“目录管理”等相近文案
3. 进入章节列表或章节编辑页面

### 成功判断

- 看到章节列表、新建章节按钮或章节编辑入口

### 失败判断

- 无法定位章节管理入口
- 页面跳转到了与章节无关的功能区

## 页面 4：章节列表页

### 目标

找到“新建章节”或已有草稿。

### 优先定位信号

- “新建章节”“新增章节”“写新章节”按钮
- 章节标题列表
- 章节状态列，如“草稿”“已发布”

### 定位策略

#### 新建草稿场景

1. 找到“新建章节”或等效按钮
2. 点击进入章节编辑器

#### 编辑已有草稿场景

1. 在章节列表中查找 `chapter_title`
2. 如果找到同名草稿，优先进入编辑而不是新建
3. 若存在重复标题，停止并报告

### 成功判断

- 已进入编辑器，或章节列表明确显示目标草稿状态

### 失败判断

- 找不到新建入口
- 找不到目标章节
- 存在重复标题且无法判断应该编辑哪一条

### 后备策略

- 如果页面上按钮文字不同，优先找“新建”“写作”“编辑”这类动作词
- 如果列表中状态列不明显，可结合标题行与操作按钮判断

## 页面 5：章节编辑器

### 目标

正确定位标题输入框和正文编辑区域。

### 优先定位信号

- 标题输入框附近有“章节标题”“标题”文案
- 正文编辑区域附近有“正文”“内容”“请输入正文”等提示
- 页面存在“保存草稿”“发布”等操作按钮

### 标题输入框定位策略

1. 优先找带“标题”“章节标题”标签的输入框
2. 如果有多个输入框，优先选择靠近正文编辑器上方的输入框
3. 输入后立刻读取当前值，确认标题已成功写入

### 正文编辑器定位策略

1. 优先找大面积可输入区域
2. 优先匹配带“正文”“内容”提示的编辑区
3. 如果是富文本或 `contenteditable` 区域，输入后必须再次读取或截图确认内容存在
4. 如果页面使用 iframe 编辑器，先进入 iframe 再操作

### 成功判断

- 标题输入框中已显示 `chapter_title`
- 正文区域中存在非空内容，并且开头段落可见

### 失败判断

- 标题输入后被自动清空
- 正文粘贴后未显示
- 正文仅显示部分内容
- 编辑器只读或不可聚焦

### 后备策略

- 如果直接输入失败，允许尝试一次粘贴式输入
- 如果富文本编辑器行为异常，停止并记录当前截图

## 页面 6：保存草稿按钮

### 目标

只点击一次保存草稿，并验证结果。

### 优先定位信号

- “保存草稿”按钮
- 等效文案，例如“保存”“存草稿”

### 定位策略

1. 优先找页面顶部或底部操作区中的“保存草稿”
2. 如果同时存在“发布”和“保存草稿”，优先选择包含“草稿”文字的按钮
3. 只点击一次

### 成功判断

满足以下任意一项：
- 页面出现“保存成功”“草稿保存成功”等提示
- 返回章节列表后能看到目标章节状态为草稿
- 页面明显保留了已保存状态

### 失败判断

- 出现错误提示
- 按钮灰掉且不可点击
- 点击后长时间没有任何状态变化

### 后备策略

- 若无明显 toast，可回章节列表验证状态
- 如果状态仍不明确，不要再次点击，直接停止并报告

## 页面 7：发布按钮与确认弹窗

### 目标

在用户明确确认后执行最终发布，并验证结果。

### 优先定位信号

- “发布”按钮
- 确认弹窗中的“确认发布”“确定”“继续发布”文案

### 定位策略

1. 发布前再次确认标题和作品名
2. 找到“发布”按钮
3. 只点击一次
4. 如果出现确认弹窗，核对文案后只确认一次

### 成功判断

满足以下任意一项：
- 出现“发布成功”提示
- 返回章节列表后显示状态为已发布
- 页面中可见已发布状态标签

### 失败判断

- 没有找到发布按钮
- 弹窗文案与预期不符
- 结果不明确且无状态变化

### 后备策略

- 如果发布后页面无提示，优先去章节列表核对
- 禁止再次点击发布来“试一试”

## 页面 8：章节状态核对页

### 目标

对比本地计划章节与平台上的真实状态。

### 优先定位信号

- 章节标题列表
- 状态标签，如“草稿”“已发布”
- 更新时间、发布时间等辅助列

### 定位策略

1. 逐项匹配标题
2. 读取每条记录对应状态
3. 记录重复标题、缺失标题、状态异常项

### 成功判断

- 能输出一份明确的状态比对结果

### 失败判断

- 无法读取章节标题
- 状态列缺失或无法识别

## 成功信号优先级

动作完成后的成功信号优先按以下顺序判断：

1. 明确成功提示文案
2. 章节状态标签变化
3. 列表中可见新增或状态更新
4. 页面标题或 URL 明显变化
5. 截图可人工确认

如果前 4 项都不存在，不要默认动作已成功。

## 禁止策略

以下做法禁止使用：
- 根据按钮位置猜测“右上角第一个就是发布”
- 仅凭颜色判断按钮功能
- 重复点击保存或发布以碰运气
- 在作品名或章节名不完全确认时继续点击

## 建议补充实测信息

真正接入 OpenClaw 前，建议根据番茄后台实际页面再补这些内容：
- 实际页面标题文案
- 章节编辑器是否为 iframe 或 `contenteditable`
- 保存成功 toast 的准确文案
- 发布确认弹窗的准确文案
- 章节列表的状态字段实际名称

这些补充越具体，浏览器代理执行时越稳定。
