# 发布记录模板

## 基本信息

- 记录时间：`YYYY-MM-DD HH:MM:SS`
- 操作人：`作者账号 / 操作人名称`
- 作品名：`作品名`
- 章节标题：`章节标题`
- 动作：`prepare_draft / review / publish / reconcile`
- 当前状态：`pending / draft_loaded / draft_saved / awaiting_confirmation / published_verified / failed_needs_review`

## 输入来源

- 源文件路径：`/home/author/novel/chapter.md`
- 章节数据文件：`chapter.example.json`
- 备注：`可选备注`

## 执行结果

- 结果：`成功 / 失败 / 部分完成`
- 结果说明：

```text
这里写本次执行的简要结果，例如：
已打开目标作品，章节内容已填入，草稿保存成功，等待人工确认发布。
```

- 警告信息：

```text
无
```

## 浏览器检查点

- 后台首页打开：`是 / 否`
- 目标作品定位成功：`是 / 否`
- 编辑器打开成功：`是 / 否`
- 标题填写成功：`是 / 否`
- 正文填写成功：`是 / 否`
- 草稿保存成功：`是 / 否`
- 发布成功：`是 / 否 / 未执行`

- 成功提示文案：`如有则填写`

## 异常与阻塞

- 是否出现登录失效：`是 / 否`
- 是否出现验证码：`是 / 否`
- 是否出现风控：`是 / 否`
- 是否出现页面结构变化：`是 / 否`
- 具体异常说明：

```text
如果失败或中断，在这里写明最后一个成功步骤，以及当前阻塞点。
```

## 后续动作

- 是否需要人工复查：`是 / 否`
- 是否允许继续发布：`是 / 否`
- 下一步建议：

```text
例如：
1. 重新登录番茄作者后台
2. 检查章节末尾分段
3. 人工确认后再执行发布
```
