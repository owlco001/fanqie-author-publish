# fanqie-author-publish

## 用途

这个技能用于通过浏览器自动化，在番茄小说作者后台完成章节的准备、检查和发布。

这个技能默认采用人工确认参与的工作流：
- 从本地文件或粘贴内容中读取章节内容
- 打开番茄作者后台并进入目标作品
- 填写章节标题和正文
- 优先保存草稿
- 在最终发布前要求用户明确确认

不要尝试无人值守登录。在执行任何发布动作前，用户应当已经在受控浏览器配置中登录好番茄作者账号。

## 适用场景

当用户需要以下操作时，使用这个技能：
- 准备番茄章节草稿
- 检查章节是否可以发布
- 在确认后发布已准备好的章节
- 核对本地章节计划与平台最近章节状态

以下情况不要使用这个技能：
- 用户还没有登录番茄作者后台
- 用户要求完全无人值守的批量发布
- 平台当前出现验证码、风控、身份校验等提示

## 输入要求

执行前需要收集或确认以下信息：
- 目标作品名
- 章节标题
- 章节正文
- 目标动作：准备草稿、检查、发布、核对
- 可选发布时间
- 可选本地源文件路径

如果缺少必要信息，只问一个能够推动下一步的短问题。

## 安全规则

- 不要要求用户在聊天里粘贴密码、短信验证码或 Cookie。
- 没有用户在当前会话里的明确确认，不要执行最终发布。
- 任何发布动作前，优先执行“保存草稿”。
- 如果页面布局、编辑器行为或关键按钮文案与预期不一致，立即停止并说明变化点。
- 如果出现验证码、风控或身份校验，立即停止并让用户手动处理。
- 在最终发布前，必须明确复述将要发布的作品和章节。
- 每执行一个关键浏览器动作后，都要先验证页面状态，再继续下一步。

## 标准流程

### 1. 准备草稿

当用户要求准备章节或保存草稿时，走这个流程。

步骤：
1. 确认目标作品、章节标题和章节正文。
2. 在浏览器工具中打开番茄作者后台。
3. 确认用户当前处于已登录状态。
4. 进入正确的作品，并打开章节编辑页。
5. 填入章节标题和正文。
6. 确认内容已经正确出现在编辑器中。
7. 保存为草稿。
8. 截图或保存页面快照用于确认。
9. 清楚报告成功或失败结果。

输出中应包含：
- 作品名
- 章节标题
- 是否已保存草稿
- 是否有警告
- 截图或状态摘要

### 2. 检查发布条件

当用户要求发布前做最终检查时，走这个流程。

检查项：
- 当前作品是否正确
- 章节标题是否存在
- 章节正文是否已加载
- 是否存在明显空字段
- 如果平台可见，草稿保存状态是否正常
- 如果平台可见，发布按钮是否存在且可点击

返回一个简短结论：
- 可以发布
- 不能发布，并说明阻塞原因

### 3. 发布章节

只有在用户已经明确确认发布时，才走这个流程。

步骤：
1. 重新说明目标作品和章节标题。
2. 如果当前会话中还没有明确确认，再次请求确认。
3. 点击最终发布动作。
4. 等待成功状态、跳转结果或确认提示。
5. 截图或保存页面快照。
6. 报告是否已验证发布成功。

如果发布结果不明确：
- 不要重复点击发布按钮
- 检查章节列表或成功提示
- 报告当前不确定状态，并询问用户如何继续

### 4. 核对最近发布状态

当用户要求把本地计划和平台状态进行比对时，走这个流程。

步骤：
1. 打开目标作品的章节列表。
2. 检查最近章节标题和状态。
3. 与提供的本地章节列表或源文件进行对比。
4. 报告以下类型的不一致：
- 缺少章节
- 标题重复
- 草稿未发布
- 出现了意外的已发布章节

## 浏览器操作原则

执行浏览器操作时：
- 优先使用稳定、可见的文字定位，不要依赖脆弱的位置假设
- 等待编辑器完全加载后再输入
- 输入或粘贴完成后，要确认内容确实出现在编辑器中，尤其是富文本编辑器
- 在关键节点截图：编辑器加载完成、草稿保存完成、发布确认完成

如果浏览器工具支持页面快照，在执行破坏性动作前优先保存快照。

## 失败处理

任何步骤失败时：
1. 立即停止流程。
2. 记录最后一个已确认成功的步骤。
3. 如果可以，立刻截图。
4. 告诉用户具体失败点，以及需要用户做什么手动处理。

常见失败场景：
- 登录失效
- 找不到目标作品
- 编辑器没有接收正文内容
- 保存草稿按钮缺失或不可点击
- 出现验证码或风控提示
- 发布确认没有出现

## 响应风格

输出要短、直接、可执行。

示例：
- `《作品名》 / 第12章 已保存草稿，可继续检查。`
- `发布被阻止：番茄后台登录已失效。`
- `《作品名》 / 第12章 已准备完成，确认后可继续发布。`

## 建议触发语句

以下请求可以视为这个技能的触发信号：
- `准备番茄章节草稿`
- `把这章发到番茄草稿箱`
- `检查这章能不能发布`
- `发布这章番茄章节`
- `核对最近几章发布状态`

## 最小执行约定

将流程状态统一视为以下之一：
- `pending`
- `draft_loaded`
- `draft_saved`
- `awaiting_confirmation`
- `published_verified`
- `failed_needs_review`

每次运行结束时，都必须明确说明当前状态。
