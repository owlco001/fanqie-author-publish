# fanqie-publisher

用于番茄小说作者后台的章节自动化发布。

## 适用场景

当用户提出以下需求时使用：

- 发布番茄小说章节
- 保存番茄草稿
- 自动上传章节正文
- 在番茄作者后台新建章节
- 批量发布网文章节
- 通过浏览器自动化操作番茄小说后台

## 输入参数

- `BOOK_ID`: 书籍 ID
- `TITLE`: 章节标题
- `BODY_PATH`: 正文文件路径
- `AUTO_PUBLISH`: 是否自动发布，`true` / `false`
- `MIN_WORDS`: 最低字数要求

## 执行原则

1. 优先尝试进入番茄章节编辑器：
   - `.../publish/?enter_from=newdraft`
   - `.../publish/?enter_from=newchapter`

2. 必须确认页面已进入编辑器：
   - 有标题输入框
   - 有正文编辑区

3. 如果未登录：
   - 不要擅自输入账号密码
   - 直接停止并汇报“需要登录”

4. 填写标题后必须验证输入框回显。

5. 填写正文后必须验证：
   - 编辑器中已存在内容
   - 页面“正文字数”已同步

6. 保存草稿后必须等待：
   - `已保存`
   - `保存成功`
   - `草稿已保存`
   等任一状态。

7. 若 `AUTO_PUBLISH=true`：
   - 继续点击发布
   - 处理确认弹窗
   - 等待终态：
     - `发布成功`
     - `已发布`
     - `审核中`
     - `待审核`
     - 或跳出 publish 页面进入章节管理页

8. 失败时必须截图并说明卡点。

## 发布前约束

在正式发布章节前：
1. 先向用户做简短确认
2. 清理当前上下文，或切换到干净执行环境
3. 再执行浏览器自动化发布

如仅保存草稿，则可直接执行。

## 输出结果

返回：

- `openedUrl`
- `currentUrl`
- `titleVerified`
- `wordCount`
- `draftSaved`
- `published`
- `publishSignal`
- `screenshots`
- `note`

## 推荐执行方式

优先使用 ACP 浏览器代理或 agent-browser 执行。
每次页面状态变化后重新 snapshot，不复用旧元素引用。

## 推荐触发词

- `发布番茄章节`
- `上传这一章到番茄`
- `保存番茄草稿`
- `在番茄后台新建章节`
- `自动发番茄`
- `把这章发到番茄小说`
